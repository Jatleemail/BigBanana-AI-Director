# Qwen 3.6 Plus 对话模型集成设计

**日期:** 2026-05-05
**状态:** 已批准

## 概述

在 BigBanana AI Director 中集成阿里云通义千问 (Qwen) 平台的 `qwen3.6-plus` 对话模型。

## API 信息

- **平台:** 阿里云 DashScope (通义千问)
- **Base URL:** `https://dashscope.aliyuncs.com`
- **端点:** `/compatible-mode/v1/chat/completions`
- **认证:** `Bearer {api_key}`
- **兼容性:** OpenAI 兼容格式，支持文本和多模态 (image_url) 输入

## 设计

### 需要修改的文件 (2个)

**1. `types/model.ts`**
- 在 `BUILTIN_PROVIDERS` 中添加 `qwen` 提供商
- 在 `BUILTIN_CHAT_MODELS` 中添加 `qwen3.6-plus` 模型定义

**2. `services/ai/apiCore.ts`**
- 在 `checkApiKey()` 中添加 Qwen 提供商专属 Key 检查，禁止回退到全局 Key

### 不需要修改的文件

- `services/adapters/chatAdapter.ts` — Qwen API 完全兼容 OpenAI 格式，现有 chat adapter 已支持通过 `endpoint` 字段自定义端点
- `services/ai/scriptService.ts` — `inferVisualStyleFromImage` 的 DeepSeek 守卫只检查 `providerId === 'deepseek'`，不会阻止 Qwen（Qwen 支持多模态）
- `components/ModelConfig/ModelCard.tsx` — 现有代码已支持显示非 AntSK 的内置模型，无需额外处理

### 关键设计决策

- **API Key 隔离:** Qwen 使用专属 DashScope API Key，不回退到全局 BigBanana Key
- **无需专用端点:** 不在模型定义中设置 `endpoint` 字段。`chatCompletion()` 和 `chatCompletionStream()` 的默认行为（`resolved?.endpoint || '/v1/chat/completions'`）已能正确处理
- **内置模型，默认启用:** 与 DeepSeek V4 Pro 保持一致的配置模式
